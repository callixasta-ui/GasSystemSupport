// features

Everything you need
to run a fuel station

From daily transactions to long-term records — all in one application, fully offline.

▣
Live Dashboard
Real-time fuel stock levels, income KPIs, monthly bar charts, and daily totals — all calculated from your data files.
⚙
Manage Transactions
Record stock received, fuel sales, credits, own use, and payments. Set fuel prices and add or remove fuel types.
📊
Reports
Monthly summaries with income vs expected charts, per-fuel breakdowns, and tabular reports for every data category.
📋
Records & Audit Trail
Every entry is editable, deletable, or undoable. Cascade handling ensures credits, sales, and payments stay consistent.
🗄
Archive System
Deleted entries are archived (effect kept). Undone entries are archived (effect reversed). Both can be restored or made permanent.
💾
Account Backup
Developer-level backup exports an account and all its data entries across every file. Import with duplicate-ID protection.
// user levels

Three levels of access

Each role sees exactly what it needs — nothing more, nothing less.

DEVELOPER
Full Control
  • Access to all admin features
  • Create, edit, and delete any account
  • Download full account backup (.devbak)
  • Extract and restore backups
  • Live account registry with auto-refresh
ADMIN
Station Manager
  • Full dashboard, manage, reports access
  • Edit/undo/delete any record from any user
  • Create and view user accounts
  • Editable timestamp, worker, and shift fields
  • Change own credentials
USER
Field Staff
  • All transaction pages (same as admin)
  • Auto-filled timestamp and worker name
  • Edit/undo/delete only own entries
  • Logs visible — others' account details masked
  • Change own credentials
// get started

Up and running
in minutes

No database setup, no server configuration. Just install and open.

1
Install Java 17+
The application requires Java 17 or later. Download it free from Adoptium or Oracle. If Java is already installed on your machine, skip this step.
2
Run the Installer
Download and run GasStationSystem_Setup_1.0.0.exe from the releases page. The installer sets up the program and creates a desktop shortcut.
3
Log In as Developer
Use the pre-installed developer account to log in for the first time. Then create admin and user accounts for your staff.
Username: dev  /  Password: Dev@1234
4
Change Default Credentials
Immediately update the developer password and 8-digit key in DevDashboard → Edit before creating any other accounts.

Common questions

Can't find an answer? Open an issue on GitHub.

No. The application is fully offline. All data is stored in plain text files on your local machine. An internet connection is only needed to download the installer initially.
All data files are stored in a data\ folder next to the application exe. Each file is a plain pipe-delimited text file — you can open and read them with Notepad if needed.
Deleting an entry keeps its effect on stock and income (the transaction happened, the numbers stay). The entry is archived in the Deleted tab and can be restored at any time. Undoing an entry reverses its effect and archives it in the Undone tab.
The application is designed as a single-machine desktop app. Multiple people can use different accounts on the same machine, but concurrent access from different computers writing to the same files simultaneously is not supported.
Log in as a developer and go to the DevDashboard. Select any account and click Download Backup. This creates a .devbak file containing the account credentials and every data entry that account created across all files. You can extract this backup on another machine to restore the data.
Java 17 or later. The application does not bundle its own Java runtime — it finds whichever JDK or JRE is already installed on your computer. If you don't have Java, download it free from Adoptium (adoptium.net) or Oracle.
